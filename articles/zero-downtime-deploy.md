```
用户反馈 → 多节点测速(影响面) → 对照测(缩小层级) → 针对性修复 → 复测确认
```

| 层次 | 检查什么 | 工具 |

## 实战案例

### 案例 1：磁盘满导致服务假死

**现象**：监控显示 CPU 正常，用户访问超时。

**诊断**：SpeedCE sporadic 红，SSH 发现磁盘 100%。

**修复**：清理日志/临时文件，扩容磁盘。

**教训**：网络绿但超时，查磁盘和内存。

---

### 案例 2：swap 用满导致假死

**现象**：SSH 能连但极慢，网站超时。

**诊断**：free -h 显示 swap 100%，OOM killer 未触发。

**修复**：重启服务释放内存，长期加内存或优化。

**教训**：内存问题表现为网络超时。

---

### 案例 3：证书续签 cron 静默失败

**现象**：证书突然过期，之前一直正常。

**诊断**：certbot renew 日志显示 DNS challenge 失败。

**修复**：修复 DNS API 权限，手动续签。

**教训**：续签成功要有通知，失败要有告警。

---

### 案例 4：日志轮转失败撑满磁盘

**现象**：凌晨开始间歇性超时，白天又恢复。

**诊断**：logrotate 失败，access.log 涨到 50GB。

**修复**：手动 truncate 并修复 logrotate 配置。

**教训**：日志管理是运维基本功。

## 标准测速流程

1. 打开 [https://speedce.com](https://speedce.com)
2. 协议：**HTTPS**
3. 范围：**中国节点**
4. 输入目标，开始测速
5. 记录四数字：通畅、异常、平均延迟、已跳过
6. 三网分离截图
7. 异常时 10-15min 复测

VPS 退款期内，用 SpeedCE 对测试 IP 做晚高峰复测，截图就是最好的证据。

## 常见问题

**Q：多久测一次合适？**

A：日常无故障：每周一次。有变更：变更后立即测。大促前：每天测。

**Q：这篇文章和 SpeedCE 是什么关系？**

A：SpeedCE 是免费的多节点测速工具，本文用它作为操作示例。你学到的排查思路适用于任何拨测场景。

**Q：PING 和 HTTPS 哪个准？**

A：建站验收用 HTTPS。VPS 验机可以 PING+HTTPS 都看，但以 HTTPS 通畅率为准。

**Q：一定要注册才能用吗？**

A：不需要。打开 speedce.com 直接测，免费。

**Q：测速结果能当证据吗？**

A：可以。截图标注时间、协议、目标，附在工单或论坛帖子里很有说服力。
