## 案例 1：DNS 迁机后新疆持续红

### 背景

迁机完成，团队都说正常，新疆同事坚持打不开。

### 排查过程

1. 打开 SpeedCE，协议 HTTPS，范围 中国节点
2. 测速结果：SpeedCE 地图：新疆持续红，其他省绿且随时间减少。
3. 定位根因并修复
4. 复测确认恢复

### 处理结果

指导 flushdns，等待 TTL 过期；必要时调低 TTL 后重新切换。

### 经验总结

迁机验收不能只看自己省份。

## 案例 2：安全组只放了 22 端口

### 背景

新购云服务器，SSH 能登，网站全国红。

### 排查过程

1. 打开 SpeedCE，协议 HTTPS，范围 中国节点
2. 测速结果：SpeedCE HTTPS 超时，PING 也超时。
3. 定位根因并修复
4. 复测确认恢复

### 处理结果

安全组添加入站 80/443，出站 443（Let's Encrypt 需要）。

### 经验总结

装机第一步：安全组，第二步：测速。

## 案例 3：Nginx 子域证书漏配

### 背景

www 正常，api 子域部分省份 HTTPS 失败。

### 排查过程

1. 打开 SpeedCE，协议 HTTPS，范围 中国节点
2. 测速结果：主域绿、API 域 sporadic 红，证书 SAN 不含 api。
3. 定位根因并修复
4. 复测确认恢复

### 处理结果

重新申请含 api 的证书，或单独配置 API server 块。

### 经验总结

每个对外子域都要单独测。

## 案例 4：证书过期导致 HTTPS 全国红

### 背景

凌晨告警，开发说服务正常，用户大面积报「连接不安全」。

### 排查过程

1. 打开 SpeedCE，协议 HTTPS，范围 中国节点
2. 测速结果：SpeedCE HTTPS 全国红、HTTP 全国绿——典型证书问题。
3. 定位根因并修复
4. 复测确认恢复

### 处理结果

检查 certbot 续签日志，手动续签后 10 分钟复测。

### 经验总结

证书自动续签失败要有告警，别等用户发现。
